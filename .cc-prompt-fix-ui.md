# UI 差异修复任务

## 目标
对比当前 UI 和 Stitch 设计稿，修复所有可见差异。

## 强制流程（按顺序执行）

### Step 1: 编译检查
```bash
npx tsc --noEmit
```

### Step 2: Build
```bash
npm run build
```

### Step 3: 截图看当前效果
```bash
bash scripts/screenshot.sh /tmp/ccdesk-after both
```
然后用 Read 工具查看：
- `/tmp/ccdesk-after/light-main.png`
- `/tmp/ccdesk-after/dark-main.png`

### Step 4: 看设计稿
用 Read 工具查看设计稿截图：
- `/tmp/stitch-new/stitch_main_chat_interface/main_chat_full_light_orange_accent_v2/screen.png`
- `/tmp/stitch-new/stitch_main_chat_interface/main_chat_desktop_dark_cn/screen.png`

### Step 5: 逐项对比并修复
对比两张图，找出差异。注意：

**已知的常见问题**：
1. 输入框 min-height 可能没生效（JS auto-resize 可能覆盖了 CSS）
2. 消息区 max-width 768px 居中可能被 flex 布局打断
3. 颜色可能用了 CSS variable 但实际渲染值有偏差
4. 间距/字号可能差几个 px

**规则**：
- 不要猜数值！用设计稿 HTML 源文件提取：`/tmp/stitch-new/.../code.html`
- 或者用 `stitch-mcp` 读取
- 修改完每个文件后从 Step 1 重新验证
- 禁止 inline style
- 禁止 width:100% 在 flex 子元素上

### Step 6: 最终验证
全部修复后，再次截图对比设计稿，确认：
- [ ] 消息布局 AI 左 / 用户右
- [ ] 输入框 min-height 100px
- [ ] 侧边栏 48px icon-only
- [ ] 发送按钮 brand-orange
- [ ] 消息区居中 max-width 768px
- [ ] 无 inline style 残留
- [ ] Dark 模式颜色正确

### Step 7: Commit
```bash
git add -A
git commit -m "fix: UI differences from Stitch design spec - [列出修复项]"
```

**记住：你必须亲眼看到截图，不能只改代码就认为完成了。**
